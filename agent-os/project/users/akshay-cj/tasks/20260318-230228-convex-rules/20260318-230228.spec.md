# Implementation Spec

## Scope

Add a dedicated Convex operating rule document and make it part of the default reading path whenever Convex is selected in a stack preset.

## Phases

1. Add `agent-os/rules/preferences/convex.md` with concise implementation and review rules.
2. Update orchestration and preference docs to reference the new rule.
3. Update Convex-enabled presets to point to the new rule and log the change in the active work item.

## Acceptance Criteria

- A dedicated Convex rule file exists in `agent-os/rules/preferences/`.
- `AGENTS.md` and `CLAUDE.md` instruct agents to read the Convex rule when Convex is part of the stack.
- `integrations.md` and Convex-enabled presets reference the new rule.
- The active work item log records the change and basic verification.
