# Implementation Spec

## Scope

Add a required documentation-impact review step for feature work and bake the same expectation into task templates.

## Documentation Impact

- Goal impact: none
- README impact: none
- Planned updates:
  - Update workflow rules to require review of `agent-os/project/goal.md` and `README.md`.
  - Update task templates so raw/spec/log files capture the documentation-impact decision.

## Phases

1. Update shared workflow rules with a documentation-impact checkpoint.
2. Update task templates to capture docs review and impact decisions.
3. Record the decision and verification in the active work-item log.

## Acceptance Criteria

- Feature workflow explicitly requires review of `agent-os/project/goal.md` and `README.md`.
- Task templates include fields for documentation-impact planning and logging.
- The active work-item log records the change and confirms whether `goal.md` and `README.md` changed.
